# 首圖背景圖片與影片製作指南

本文件用於規劃網站分頁導覽上方的首圖區塊（Hero Banner）背景。目標是在保留目前文字、聯絡卡片與日系編輯風格的前提下，增加東京生活與不動產的視覺氛圍。

## 建議優先順序

第一版建議先使用「靜態背景圖片＋網頁遮罩」。

靜態圖片載入速度較快、手機顯示較穩定，也不會干擾標題閱讀。影片可在圖片版本確認構圖與遮罩後，再作為第二階段升級。

## 推薦視覺方向

- 東京住宅街、低層公寓、住宅車站周邊或自然採光的居家建築。
- 氣氛以清晨、柔和日光、安定生活感為主。
- 色調可使用暖白、灰綠、淺灰，配合網站主色 `#00A174`。
- 畫面可帶一點淺景深，降低背景對文字的干擾。
- 避免大量人臉、車牌、密集招牌、過度雜亂的電線或強烈廣告文字。
- 不建議使用東京鐵塔、澀谷十字路口等觀光感太強的素材，以免偏離「在日本生活與找房」的品牌方向。

## 背景分層規劃

建議由下至上分成四層：

1. 備援底色：`#F5F8F6`。
2. 背景圖片或影片：滿版填滿首圖區域。
3. 漸層遮罩：確保標題與說明文字清晰。
4. 前景內容：標題、介紹文字、聯絡卡片與按鈕。

概念如下：

```text
標題、介紹文字與聯絡卡片
────────────────────
白色／灰綠色半透明漸層遮罩
────────────────────
低飽和背景圖片或背景影片
────────────────────
#F5F8F6 備援底色
```

## 桌面版圖片規格

- 建議製作尺寸：`2400 × 1000 px`。
- 可接受比例：約 `2.4:1`。
- 原始檔可提供 PNG、JPG 或 WebP。
- 網頁正式使用時優先轉為 AVIF，並保留 WebP 備援。
- 正式桌面版圖片建議控制在約 `300 KB` 內。
- 重要主體避免放在左側，因為左側會放置主要標題與介紹文字。
- 不需預先把白色遮罩、漸層或文字做進圖片，這些會由網頁另外疊加，方便之後調整。
- 請保留高畫質原始檔，不要只保留壓縮後版本。

## 手機版圖片規格

- 建議另外製作：`1200 × 1400 px`。
- 正式手機版圖片建議控制在約 `180 KB` 內。
- 不建議直接把桌面橫圖硬裁成直圖，重要建築或視覺焦點可能會消失。
- 手機版應讓畫面主體位於中間或中下區域，避免被標題與聯絡內容遮住。
- 若暫時只有桌面圖，第一版可先使用 CSS 調整 `object-position`，之後再補手機專用裁切。

## 桌面版遮罩建議

左側文字區維持較高白色濃度，往右側逐漸露出背景：

```css
background: linear-gradient(
  90deg,
  rgba(255, 255, 255, 0.96) 0%,
  rgba(255, 255, 255, 0.88) 48%,
  rgba(245, 248, 246, 0.68) 100%
);
```

實際濃度會依照片明暗調整，不會固定套用同一組數字。

## 手機版遮罩建議

手機版改用上下方向遮罩，讓文字在不同裁切下仍可閱讀：

```css
background: linear-gradient(
  180deg,
  rgba(255, 255, 255, 0.92) 0%,
  rgba(255, 255, 255, 0.82) 100%
);
```

## 圖片顯示與可讀性

- 圖片以 `object-fit: cover` 填滿區域。
- 桌面與手機需分別調整 `object-position`，避免重要主體被裁掉。
- 背景可降低約 10%～25% 飽和度，讓網站主色與文字更穩定。
- 如照片細節過多，可稍微降低對比或加入極輕微模糊，但不應讓畫面失去質感。
- 深色文字與背景的對比需至少達到一般文字可閱讀標準；必要時提高遮罩濃度。
- 聯絡卡片可維持白色，或改為約 92%～96% 不透明白底搭配輕微背景模糊。

## 影片版建議規格

- 建議長度：8～12 秒。
- 建議尺寸：`1920 × 1080 px`。
- 影格率：24 或 30 fps。
- 主要格式：H.264 MP4；可另外提供 WebM。
- 正式影片建議控制在約 `3 MB` 內，最高不宜超過 `4 MB`。
- 必須靜音，不可自動播放聲音。
- 建議使用緩慢鏡頭、固定鏡位或非常輕微的移動。
- 避免快速運鏡、閃爍燈光、明顯跳接、大量行人與可辨識人臉。
- 循環首尾應自然銜接，避免每次重新播放時明顯跳動。
- 影片本身不要加入文字、Logo 或按鈕。

網頁播放設定建議：

```html
<video autoplay muted loop playsinline preload="metadata" poster="/hero-poster.webp">
  <source src="/hero-background.webm" type="video/webm" />
  <source src="/hero-background.mp4" type="video/mp4" />
</video>
```

## 影片效能與無障礙處理

- 必須準備靜態 poster 圖，避免影片載入前出現黑畫面。
- 手機版建議優先顯示靜態圖片，避免耗用過多流量與電量。
- 使用者開啟「減少動態效果」時，應停止影片並改顯示靜態圖。
- 使用者開啟省流量模式時，應避免自動載入完整影片。
- 影片載入失敗時，仍須保留背景圖片與備援底色。
- 影片不能影響首圖文字、聯絡按鈕或分頁導覽的操作。

## 推薦的第一版成品

建議先使用以下組合：

- 東京清晨住宅街或質感公寓外觀。
- 照片帶有自然光與輕微景深。
- 畫面降低約 20% 飽和度。
- 左側覆蓋較濃白色漸層，保護標題與介紹文字。
- 右側保留淡淡的建築或街景細節。
- 聯絡卡片維持清楚的白色層次。
- 手機版另外調整裁切與遮罩方向。

## 提供素材後的實作流程

1. 將原始圖片或影片放入專案素材資料夾。
2. 產生桌面與手機版本。
3. 壓縮為正式網頁格式。
4. 加入背景媒體、備援色與遮罩層。
5. 調整桌面與手機裁切位置。
6. 確認標題、介紹文字與聯絡卡片的對比。
7. 檢查不同螢幕尺寸、載入速度與減少動態效果設定。
8. 確認視覺後再決定是否使用影片版。

## 素材交付清單

圖片版本至少提供：

- 桌面原始圖：建議 `2400 × 1000 px`。
- 高畫質未壓縮版本。
- 若已製作，另附手機直式裁切圖。

影片版本至少提供：

- 未加文字的原始影片。
- 建議 8～12 秒循環版本。
- 一張可作為 poster 的代表畫面。

素材完成後，先在本機套用並確認視覺，不會在未確認前自行 commit 或 push。
