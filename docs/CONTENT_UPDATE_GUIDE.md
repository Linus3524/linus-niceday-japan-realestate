# 網站知識內容更新指南

本網站的租屋、買房與 AI 回答使用同一份知識資料。日後需要更新時，直接在對話中告訴 Codex 要更新的主題即可，例如：

- 「更新 2026 年第四季東京租金行情」
- 「查核並更新某家銀行的海外買方貸款條件」
- 「更新中央區民泊規定」
- 「新增一題關於退租清算的 Q&A」

Codex 的標準更新流程：

1. 確認資訊類型：法令／官方指引、金融方案、一般實務或市場概算。
2. 對會變動的內容查核官方或第一手來源；法規、金融與住宿營業資訊不得只憑舊文案更新。
3. 修改共用知識來源：`src/data/rentGuideData.ts` 或 `src/data/buyHouseData.ts`。
4. 視需要調整頁面說明、試算器、AI 規則與官方來源連結。
5. 更新對應資料的 `reviewedAt` 日期。
6. 在 `docs/CONTENT_CHANGELOG.md` 記錄更新日期、範圍、資料性質與來源。
7. 執行 `npm run lint` 與 `npm run build`；除非另有指示，不會自行 commit 或 push。

## 非公開資料、截圖與社群貼文

銀行專員通知、仲介內網資料、管理公司信件、小紅書／微博／社群貼文、照片或聊天紀錄，都可以直接提供給 Codex。最理想的形式是原始連結、完整截圖、PDF、電子郵件全文，或標示取得日期與發布者的照片。

這類資料不會被直接寫成「官方規則」。Codex 會在網站資料中標示為下列其中一種狀態：

- `官方來源已核驗`：可連到政府、自治體、金融機構或正式文件的原始來源。
- `使用者提供，待交叉核驗`：資料有參考價值，但尚無法透過公開正式來源確認。
- `待提供官方或書面來源`：現有內容未取得足夠證據，不應作為客戶決策依據。

若社群貼文提供的是現場公告、銀行窗口回覆或管理公司通知，可作為追查線索；在無法確認原始文件、日期、地區與適用條件前，網站會保留為實務參考，不會宣稱適用所有案件。

## 建議檢視節奏

每季（1、4、7、10 月）檢視下列資訊；有制度變動或使用者指定時可隨時更新。

| 類型 | 主要位置 | 更新原則 |
| --- | --- | --- |
| 租金與買房概算 | `src/data/*Data.ts`、`CalculatorTab.tsx` | 檢查基準資料、計算假設與資料期間 |
| 銀行方案 | `src/data/buyHouseData.ts` | 以銀行最新公開資料或書面方案為準 |
| 民泊／旅館業 | `src/data/buyHouseData.ts` | 以自治體、保健所、觀光廳或國土交通省公告為準 |
| 租屋法規與費用 | `src/data/rentGuideData.ts` | 以國土交通省及契約／募集條件為準 |
| AI 回答規則 | `api/chat.ts` | 確保 AI 對過期或個案資訊保留條件與確認提醒 |

## 資料治理原則

- 法令、自治體規則、銀行利率與審查條件：必須附來源與最後檢視日期。
- 租金、房價與試算係數：標示為概算，不得描述為保證成交價或銀行鑑價。
- 契約內容：以個別物件的書面條款為準。
- 仲介實務經驗：清楚標示為一般實務，不可寫成所有案件都適用。
- 網頁與 AI 應維持使用同一份知識資料，避免答案與頁面內容不同步。
